📋 Step-by-Step Azure App Registration Guide

1

Go to Azure Portal

Sign in to portal.azure.com with your Microsoft 365 Global Administrator account.

Navigate to: Microsoft Entra ID (Azure Active Directory) → App registrations → New registration

2

Register the Application

FieldValue to Enter
NameSmartShield CyberAware
Supported account typesAccounts in this organizational directory only (Single tenant)
Redirect URISingle-page application (SPA) →
3

Copy Your IDs

After registration, on the Overview page, copy:

  • Application (client) ID → Paste in Configuration tab → Client ID field
  • Directory (tenant) ID → Paste in Configuration tab → Tenant ID field
4

Configure Authentication (MSAL Token Settings)

Go to Authentication tab in your App registration:

  • Under Single-page application — verify the redirect URI is added
  • Enable Access tokens and ID tokens under Implicit grant
  • Set Supported account types to your organization only
5

Add API Permissions

Go to API permissions → Add a permission → Microsoft Graph → Delegated permissions. Add:

openid
profile
email
User.Read
User.ReadBasic.All
Mail.Send
⚠️Click "Grant admin consent for [Organization Name]" — required for User.ReadBasic.All.
6

Test the Connection

Return to the Configuration tab, enter your Client ID and Tenant ID, save, then click "Test Connection". A Microsoft login popup will appear.

✅ Once connected, you can: Sign in with Microsoft on the login page · Sync users from Azure AD · Send phishing simulation emails · Send training reminders via Office 365.

👥 Sync Users from Azure AD

Import employees directly from your Microsoft 365 / Azure Active Directory. New users are added to the platform; existing users (matched by email) are updated.

⚠️You must sign in with Microsoft to sync users from Azure AD.

ℹ️ Sync Behavior

ScenarioAction
New user in Azure AD (not in platform)✅ Created as Employee
Existing user matched by email🔄 Updated (name, dept, title)
Disabled account in Azure AD⚠️ Marked Inactive
Service/system accounts (#EXT#)⏭ Skipped
Admin accounts✅ Created (still as Employee role)
ℹ️ Synced users are given a default password of AzureAD_SSO. They should log in using Sign in with Microsoft instead. Department groupings are automatically created based on Azure AD department attribute.
